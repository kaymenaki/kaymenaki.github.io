Posted by Eirini Kaymenaki on May 5, 2020

I am interested in doing some research and analysis in the use of domain specific languages in the user experience field. In particular, my idea is around user journey design and mapping and how we can model, analyse and visualise dynamic user journeys with DSL methodology and approach.

Thesis Outline

  1. Background
    • User journey history and background (how did it come up, why, and where else do we find them)
    • User journeys variants (identify and synthesise)
      • Concepts and notations (visual/representation)
      • Related notations (sequence diagrams) in other fields
  2. Analysis
    • Open challenges and opportunities
  3. Domain specific language design and implementation
    • Concepts (meta-model)
    • Visual notation
    • Interactivity elements
    • Implementation
  4. Evaluation
    • Case studies (use the language to implement existing user journeys, collected early in the project but not used for development purposes)
    • Completeness
    • Expressiveness
    • Domain fitness
